## Overview

This step wires euler_txn to call connector-service instead of directly calling the gateway. euler_txn remains the orchestrator -- it handles auth, DB state, routing, and async workflows. Only the HTTP call to the gateway is delegated.

**Key insight: the bridge supports per-flow granularity.** You can migrate a single flow (e.g., only `getSdkParams`) for a gateway without migrating other flows (e.g., `sendCollectRequest`, refund, sync). Each flow is independently toggleable via Redis config.

## Step 2a: Understand the Feature Flag System

### The Gating Function

**File:** `euler-x/src-generated/Utils/Cutover.hs` (lines 196-205)

```haskell
isConnectorServiceEnabledForFunction :: Text -> Text -> Maybe Text -> BackendFlow st Configs Bool
isConnectorServiceEnabledForFunction functionName gateway mid = do
  optConf <- findByNameFromRedis (C.connectorServiceCutover gateway)
  case optConf of
    Nothing -> pure False
    Just conf -> do
      let functionsEnabled = fromMaybe [] (conf.enabledFunctions)
          isFunctionEnabled = functionName `elem` functionsEnabled
      isMerchantEnabled <- maybe (pure False) (isFeatureEnabledOnConf conf.enabledMerchants) mid
      pure $ isFunctionEnabled && isMerchantEnabled
```

**Parameters:**
- `functionName` — the specific flow: `"getSdkParams"`, `"sendCollectRequest"`, or a new flow name you add
- `gateway` — the gateway name string: `"RAZORPAY"`, `"PHONEPE"`, `"HDFC"`, etc.
- `mid` — merchant ID (for per-merchant rollout)

### The Config Structure

**File:** `euler-x/src-generated/Utils/Cutover.hs` (lines 166-169)

```haskell
data ConnectorServiceCutoverConfig = ConnectorServiceCutoverConfig
  { enabledFunctions :: Maybe [Text]   -- list of flow names enabled for this gateway
  , enabledMerchants :: FeatureConf    -- per-merchant rollout config
  }
```

### The Redis Key

**File:** `euler-x/src-generated/Config/Constants.hs` (lines 870-875)

```haskell
connectorServiceCutover gateway = CONNECTOR_SERVICE_CUTOVER gateway
-- Redis key: "CONNECTOR_SERVICE_CUTOVER_<GATEWAY>"
-- e.g., "CONNECTOR_SERVICE_CUTOVER_HDFC"
```

### Granularity Summary

| Dimension | Mechanism | Example |
|-----------|-----------|---------|
| **Per-gateway** | Redis key `CONNECTOR_SERVICE_CUTOVER_<GATEWAY>` | Key exists = gateway is configured |
| **Per-flow** | `enabledFunctions` list in the config value | `["getSdkParams"]` = only authorize is bridged |
| **Per-merchant** | `enabledMerchants` in the config value | Supports rollout %, enable-all, disable-any |

### Per-Gateway Guard (Razorpay-Specific)

**File:** `euler-x/src-generated/Gateway/ConnectorService/Flow.hs` (lines 406-416)

```haskell
getFlowEnabledForUcs gateway txn =
  case gateway of
    PG.RAZORPAY -> handleRzpForUcs  -- blocks TPV_PAYMENT + S2S-disabled merchants
    PG.UNIPG    -> handleRzpForUcs
    _           -> pure True        -- all other gateways: defer to cutover config
```

For non-Razorpay gateways, this always returns `True`, so the cutover config is the only gate.

## Step 2b: Add Gateway to Bridge Dispatcher

**File:** `euler-x/src-generated/Gateway/ConnectorService/Flow.hs`

For each flow you want to bridge, the dispatcher routes to gateway-specific handlers.

### For `getSdkParams` (Authorize)

The call site is in `CommonGateway.hs`:

```haskell
-- CommonGateway.hs (around line 2085)
useConnectorService <- Cutover.isConnectorServiceEnabledForFunction
  "getSdkParams" (show gateway) (txn ^. L._merchantId)
isFlowEnabledForUcs <- ConnectorService.getFlowEnabledForUcs gateway txn

if useConnectorService && isFlowEnabledForUcs
  then ConnectorService.callConnectorServiceGetSdkParams gateway ord txn mga ...
  else -- old path: call gateway adapter directly
```

In `ConnectorService/Flow.hs`, the `callConnectorServiceGetSdkParams` function dispatches to per-gateway handlers:

```haskell
callConnectorServiceGetSdkParams gateway ... = do
  (reqBody, url, headers) <- case gateway of
    PG.RAZORPAY -> buildRazorpayReqAndHeaders ...
    PG.PHONEPE  -> buildPhonePeReqAndHeaders ...
    PG.<NEW_GW> -> build<NewGw>ReqAndHeaders ...    -- ADD YOUR GATEWAY HERE
    _ -> throwErr GATEWAY_NOT_FOUND_OR_NOT_SUPPORTED
  makeConnectorServiceRequest url headers reqBody
```

### For New Flows (Refund, Sync, etc.)

To bridge a flow that isn't wired yet (e.g., Refund), you need to:

1. **Add a new function** in `ConnectorService/Flow.hs` (e.g., `callConnectorServiceRefund`)
2. **Add the call site** in `CommonGateway.hs` (or the relevant orchestration module) with the `isConnectorServiceEnabledForFunction` gate using a new function name (e.g., `"createRefund"`)
3. **Add the function name** to the Redis cutover config's `enabledFunctions` list

### Response Converter

Map `ConnectorServiceSdkResponse` back to euler_txn's types:

```haskell
convert<GatewayName>Response :: ConnectorServiceSdkResponse -> TxnDetail -> OrderReference
  -> BackendFlow st Configs PaymentResp
convert<GatewayName>Response csResponse txnDetail orderRef = do
  -- 1. Extract gateway transaction ID
  let epgTxnId = case csResponse.response_ref_id of
        Just ref -> Just (ref.id_type.id)
        Nothing  -> Nothing

  -- 2. Update epgTxnId in DB
  _ <- SF.updateEpgTxnIdAndGatewayAuthReqParamsOpt
    txnDetailId epgTxnId (Just $ encodeJSON csResponse)

  -- 3. Map status (CRITICAL: must match old path behavior exactly)
  let status = case T.toUpper <$> csResponse.status of
        Just "AUTHORIZED"              -> Right AUTHORIZED
        Just "CHARGED"                 -> Right CHARGED
        Just "AUTHENTICATION_PENDING"  -> Right PENDING_VBV
        Just "PENDING"                 -> Right AUTHORIZING
        Just "FAILURE"                 -> Left $ fromMaybe "UNKNOWN" csResponse.error_code
        _                              -> Left "UNKNOWN_STATUS"

  -- 4. Build PaymentResp based on flow type
  case status of
    Right txnStatus -> buildSuccessPaymentResp csResponse txnStatus txnDetail orderRef
    Left errorCode  -> buildErrorPaymentResp errorCode csResponse txnDetail
```

## Step 2d: Configure the Feature Flag

Set the Redis key to enable your flow:

```json
// Redis key: CONNECTOR_SERVICE_CUTOVER_<GATEWAY>
// Example for HDFC with only getSdkParams enabled:
{
  "enabledFunctions": ["getSdkParams"],
  "enabledMerchants": {
    "enableAll": false,
    "enabledList": ["merchant_123", "merchant_456"],
    "disableAny": []
  }
}
```

**To enable additional flows later**, just add them to `enabledFunctions`:

```json
{
  "enabledFunctions": ["getSdkParams", "sendCollectRequest", "createRefund"],
  "enabledMerchants": { "enableAll": true, "disableAny": [] }
}
```

**To disable a specific flow**, remove it from the list. The old path will be used automatically.

## Migrating a Single Flow

Because the feature flag is per-flow, you can migrate just one flow at a time:

1. **Implement only that flow** in connector-service (e.g., only `Authorize`)
2. **Use `macro_connector_implementation!`** for all other flows
3. **Wire only that flow's call site** in `CommonGateway.hs`
4. **Set `enabledFunctions` to just that flow** in Redis config
5. **Roll out** using the merchant rollout config
6. **Later**, add more flows to the connector and to `enabledFunctions`

This is the safest migration path — one flow at a time, fully reversible.

## Step 2e: Wire the HTTP Call

The actual HTTP call to connector-service uses the existing pattern in `ConnectorService/Flow.hs`:

```haskell
makeConnectorServiceRequest :: Text -> [(Text, Text)] -> RequestBody -> ApiTag
  -> BackendFlow st Configs ConnectorServiceSdkResponse
makeConnectorServiceRequest connectorServiceUrl httpHeaders reqBody apiTag = do
  let jsonBody = A.encode reqBody
  let httpRequest = HTTPRequest
        { getRequestMethod = Post
        , getRequestHeaders = httpHeaders
        , getRequestBody = Just (LBinaryString jsonBody)
        , getRequestURL = connectorServiceUrl
        , getRequestTimeout = Just 30000  -- 30s timeout
        }
  response <- lift . lift $ L.callHTTPWithManager' (Just "tlsManager") httpRequest apiTag
  case A.decode (getResponseBody response) of
    Just csResponse -> pure csResponse
    Nothing -> throwErr CONNECTOR_SERVICE_PARSE_ERROR
```
