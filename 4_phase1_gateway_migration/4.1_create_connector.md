## Overview

This step creates or updates the connector adapter in `connector-service` (Rust). The connector handles:
- Building gateway-specific HTTP requests
- Making API calls to the gateway
- Parsing gateway responses into unified types
- Mapping gateway statuses to `AttemptStatus`

---

## File Structure

Create these files in `backend/connector-integration/src/connectors/`:

```
connectors/
└── <connector_name>/
    ├── mod.rs              # ConnectorCommon + flow trait implementations
    └── transformers.rs     # Request/response type conversions
```

## Step 1a: Define the Connector Struct + ConnectorCommon

**File:** `connectors/<connector_name>/mod.rs`

**Source to read:** euler_txn's `Gateway/<Name>/Flow.hs` -- look for base URL, auth header construction, content type, error parsing.

```rust
pub mod transformers;

use common_utils::{request::RequestContent, CustomResult};
use domain_types::{
    errors::ConnectorError,
    router_data::ErrorResponse,
    router_data_v2::RouterDataV2,
};
use hyperswitch_masking::Maskable;

use crate::api;

#[derive(Debug, Clone)]
pub struct <ConnectorName>;

impl api::ConnectorCommon for <ConnectorName> {
    fn id(&self) -> &'static str {
        "<connector_name>"
    }

    fn base_url(&self, connectors: &Connectors) -> String {
        // Map from euler_txn: look in Gateway/<Name>/Flow.hs for the base URL
        // Usually constructed from MerchantGatewayAccount or a constant
        connectors.<connector_name>.base_url.clone()
    }

    fn get_auth_header(
        &self,
        auth_type: &ConnectorAuthType,
    ) -> CustomResult<Vec<(String, Maskable<String>)>, ConnectorError> {
        // Map from euler_txn: look in Gateway/<Name>/Flow.hs for header construction
        // euler_txn reads from MerchantGatewayAccount fields:
        //   apiKey, key1, merchantId, etc.
        //
        // Map to ConnectorAuthType variants:
        //   HeaderKey { api_key }
        //   BodyKey { api_key, key1 }
        //   SignatureKey { api_key, key1, api_secret }
        //
        // See: 7.1_type_mapping.md -> Auth Type Mapping
    }

    fn common_get_content_type(&self) -> &'static str {
        // Most gateways: "application/json"
        // Some gateways: "application/x-www-form-urlencoded"
        // XML gateways: "application/xml"
        "application/json"
    }

    fn get_error_response(
        &self,
        res: Response,
    ) -> CustomResult<ErrorResponse, ConnectorError> {
        // Map from euler_txn: look for error parsing in Gateway/<Name>/Flow.hs
        // euler_txn typically extracts bankErrorCode + bankErrorMessage
        //
        // Map to:
        // ErrorResponse {
        //     status_code: res.status_code,
        //     code: <bankErrorCode>,
        //     message: <bankErrorMessage>,
        //     reason: <additional detail>,
        //     attempt_status: None,
        // }
    }
}
```

## Step 1b: Implement ConnectorIntegrationV2 for Each Flow

For each flow the gateway supports, implement the trait. Below is the Authorize flow as a template.

**File:** `connectors/<connector_name>/mod.rs` (same file, below ConnectorCommon)

**Source to read:** euler_txn's `Gateway/<Name>/Flow.hs` -- `initiateTxn` function.

```rust
impl ConnectorIntegrationV2<
    connector_flow::Authorize,
    PaymentFlowData,
    PaymentsAuthorizeData,
    PaymentsResponseData,
> for <ConnectorName> {

    fn get_headers(
        &self,
        req: &RouterDataV2<connector_flow::Authorize, PaymentFlowData, PaymentsAuthorizeData, PaymentsResponseData>,
    ) -> CustomResult<Vec<(String, Maskable<String>)>, ConnectorError> {
        // Source: Gateway/<Name>/Flow.hs -- HTTP header construction
        // Common patterns:
        //   Authorization: Bearer <api_key>
        //   X-Merchant-Id: <merchant_id>
        //   Content-Type: application/json
        self.build_headers(req, self.get_auth_header(&req.connector_config.connector_auth)?)
    }

    fn get_url(
        &self,
        req: &RouterDataV2<...>,
    ) -> CustomResult<String, ConnectorError> {
        // Source: Gateway/<Name>/Transforms.hs -- URL construction
        // Pattern: base_url + "/api/v1/payments" or similar
        Ok(format!(
            "{}/api/v1/payments",
            self.base_url(req.resource_common_data.connectors)?
        ))
    }

    fn get_request_body(
        &self,
        req: &RouterDataV2<...>,
    ) -> CustomResult<Option<RequestContent>, ConnectorError> {
        // Delegate to transformers.rs
        let body = transformers::<ConnectorName>PaymentsRequest::try_from(req)?;
        Ok(Some(RequestContent::Json(Box::new(body))))
    }

    fn handle_response_v2(
        &self,
        data: &RouterDataV2<...>,
        res: Response,
    ) -> CustomResult<RouterDataV2<...>, ConnectorError> {
        // Source: Gateway/<Name>/Flow.hs -- response parsing + status mapping
        // CRITICAL: Status mapping must match euler_txn's existing behavior exactly
        let response: transformers::<ConnectorName>PaymentsResponse =
            res.response.parse_struct("<ConnectorName>PaymentsResponse")?;

        // Map to unified response using ResponseRouterData
        // IMPORTANT: ResponseRouterData has exactly 2 type parameters:
        //   ResponseRouterData<ConnectorResponseType, RouterDataType>
        // where RouterDataType = Self (i.e., RouterDataV2<Flow, CommonData, Req, Resp>)
        RouterDataV2::try_from(ResponseRouterData {
            response,
            router_data: data.clone(),
            http_code: res.status_code,
        })
    }
}
```

**Repeat for each flow:**

| Flow | Trait | euler_txn Source Function |
|------|-------|-------------------------|
| PSync | `ConnectorIntegrationV2<PSync, PaymentFlowData, PaymentsSyncData, PaymentsResponseData>` | `syncTxnStatus` in Flow.hs |
| Capture | `ConnectorIntegrationV2<Capture, PaymentFlowData, PaymentsCaptureData, PaymentsResponseData>` | `processCapture` in Flow.hs |
| Void | `ConnectorIntegrationV2<Void, PaymentFlowData, PaymentVoidData, PaymentsResponseData>` | `processVoid` in Flow.hs |
| Refund | `ConnectorIntegrationV2<Refund, RefundFlowData, RefundsData, RefundsResponseData>` | `make_<Gw>RefundRequest` in Transforms.hs |
| RSync | `ConnectorIntegrationV2<RSync, RefundFlowData, RefundSyncData, RefundsResponseData>` | `syncRefundStatus` in Flow.hs |

## Step 1c: Implement Transformers

**File:** `connectors/<connector_name>/transformers.rs`

**Source to read:** euler_txn's `Gateway/<Name>/Transforms.hs` and `Gateway/<Name>/Types.hs`.

### Request Transformer

```rust
use serde::Serialize;

// Map from euler_txn's make_<Gateway>AuthRequest / make_<Gateway>EnrollmentRequest
#[derive(Debug, Serialize)]
pub struct <ConnectorName>PaymentsRequest {
    // Fields from euler_txn's gateway-specific request type
    // Read Gateway/<Name>/Types.hs for the exact field names
    pub merchant_id: String,
    pub amount: String,
    pub currency: String,
    pub order_id: String,
    pub customer_email: Option<String>,
    pub return_url: String,
    // ... gateway-specific fields
}

impl TryFrom<&RouterDataV2<
    connector_flow::Authorize,
    PaymentFlowData,
    PaymentsAuthorizeData,
    PaymentsResponseData,
>> for <ConnectorName>PaymentsRequest {
    type Error = error_stack::Report<ConnectorError>;

    fn try_from(item: &RouterDataV2<...>) -> Result<Self, Self::Error> {
        // Translate euler_txn's request construction logic
        // Source: Gateway/<Name>/Transforms.hs -- make_<Gateway>AuthRequest function
        //
        // euler_txn reads from:
        //   OrderReference (amount, currency, orderId, customerEmail)
        //   TxnDetail (txnId, merchantId, gateway)
        //   MerchantGatewayAccount (apiKey, merchantId, extra config)
        //   TxnCardInfo (cardNumber, cardExpiry, cvv, cardType)
        //
        // connector-service reads from:
        //   item.request (PaymentsAuthorizeData -- amount, currency, payment_method_data)
        //   item.resource_common_data (PaymentFlowData -- merchant_id, etc.)
        //   item.connector_config (ConnectorSpecificConfig -- auth, metadata)

        Ok(Self {
            merchant_id: item.resource_common_data.merchant_id.clone(),
            amount: item.request.amount.to_string(),
            currency: item.request.currency.to_string(),
            order_id: item.request.order_id.clone(),
            customer_email: item.request.email.clone(),
            return_url: item.request.return_url.clone().unwrap_or_default(),
        })
    }
}
```

use serde::Deserialize;

// Map from euler_txn's <Gateway>AuthorizeResponse / <Gateway>TxnStatusSyncResponseData
#[derive(Debug, Deserialize)]
pub struct <ConnectorName>PaymentsResponse {
    // Fields from euler_txn's gateway-specific response type

    pub transaction_id: Option<String>,
    pub error_code: Option<String>,
    pub error_message: Option<String>,
    pub redirect_url: Option<String>,
    // ... gateway-specific fields
}

// Status enum for this gateway
#[derive(Debug, Deserialize)]
#[serde(rename_all = "SCREAMING_SNAKE_CASE")]
pub enum <ConnectorName>PaymentStatus {
    Success,
    Pending,
    Failed,
    Cancelled,
    // ... gateway-specific statuses
}

// CRITICAL: This mapping must exactly match euler_txn's behavior
// Source: Gateway/<Name>/Flow.hs -- look for status mapping logic
impl From<<ConnectorName>PaymentStatus> for AttemptStatus {
    fn from(status: <ConnectorName>PaymentStatus) -> Self {
        match status {
            <ConnectorName>PaymentStatus::Success   => Self::Charged,
            <ConnectorName>PaymentStatus::Pending   => Self::Pending,
            <ConnectorName>PaymentStatus::Failed    => Self::Failure,
            <ConnectorName>PaymentStatus::Cancelled => Self::Voided,
        }
    }
}

## Step 1e: Mark Unsupported Flows

For flows the gateway does NOT support, use the macro:

```rust
// In mod.rs
macro_connector_implementation!(
    <ConnectorName>,
    connector_flow::Void,
    connector_flow::RSync,
    connector_flow::AcceptDispute,
    connector_flow::DefendDispute,
    connector_flow::SubmitEvidence,
    // ... list all unsupported flows
);
```

This generates empty trait implementations that return "not supported" errors.
