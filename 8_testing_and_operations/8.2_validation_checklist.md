# 8.2 Validation Checklist

> **Testing & Operations — Per Gateway, Per Flow Sign-Off**
>
> Complete this checklist for **every gateway x flow combination** before promoting
> traffic to the new path. Copy and fill out for each migration.

---

## Checklist Template

### Gateway: `______________` | Flow: `______________`

#### Request Parity

- [ ] connector-service sends the **exact same HTTP request** to the gateway as euler_txn would
  - [ ] HTTP method matches (POST/GET/PUT)
  - [ ] URL matches (path, query parameters)
  - [ ] Headers match (auth headers, content-type, custom gateway headers)
  - [ ] Body format matches (JSON/form-encoded/XML)
  - [ ] Body field names match (exact casing, no extra/missing fields)
  - [ ] Body field values match (amount format, currency format, encoding)

#### Response Mapping

- [ ] All gateway response statuses map to the correct `AttemptStatus`
  - [ ] Success status -> correct terminal state
  - [ ] Failure status -> correct failure state
  - [ ] Pending status -> correct intermediate state
  - [ ] Redirect status -> correct redirect handling
  - [ ] Unknown/unexpected status -> safe default behavior

#### Error Handling

- [ ] Gateway errors produce the same `bankErrorCode`/`bankErrorMessage` values
- [ ] HTTP 4xx errors are handled (bad request, unauthorized, not found)
- [ ] HTTP 5xx errors are handled (gateway server error, gateway timeout)
- [ ] Malformed response body does not cause panic/crash
- [ ] Connection timeout produces appropriate error response
- [ ] Response timeout produces appropriate error response

#### Authentication

- [ ] Gateway authentication works (API key, signature, certificate)
- [ ] Auth credentials are correctly extracted from `MerchantGatewayAccount` -> gRPC metadata
- [ ] Auth type (HeaderKey, BodyKey, SignatureKey) is correctly handled

#### Encryption

- [ ] If the gateway requires request encryption (e.g., HDFC AES, JWE), it is correctly implemented
- [ ] If the gateway requires response decryption, it is correctly implemented
- [ ] If the gateway uses request signing (HMAC, RSA), it is correctly implemented

#### Redirect Flows

- [ ] Redirect URLs are correctly constructed and returned
- [ ] Form POST data (if any) is correctly formatted
- [ ] Callback/return URL handling works

#### Webhook Parsing

- [ ] Incoming webhooks from the gateway are correctly parsed
- [ ] Webhook signature/authentication is verified
- [ ] Webhook event types are correctly mapped

#### Edge Cases

- [ ] Zero-amount auth (e.g., card verification)
- [ ] Partial capture (if applicable)
- [ ] Multi-currency handling
- [ ] International cards (if applicable)
- [ ] Special characters in customer name/email
- [ ] Very large amounts
- [ ] Very small amounts (below minimum)
- [ ] Idempotency — retry-safe; duplicate requests don't cause double charges

#### Performance

- [ ] p50 latency within 10ms of old path
- [ ] p99 latency within 50ms of old path
- [ ] No increase in gateway error rate vs old path

#### Timeout Handling

- [ ] HTTP timeouts produce appropriate error responses
- [ ] Timeout values match euler_txn's gateway-specific timeout configuration

---

## How to Use This Checklist

1. **Copy the template** for each gateway x flow combination (e.g., "HDFC x Authorize", "HDFC x Capture", "HDFC x Refund")
2. **Check off items** as you verify each one
3. **Document evidence** — note the test case or log entry that confirms each item
4. **Get sign-off** from a reviewer before promoting traffic

### Example: Filled Checklist

```
Gateway: HDFC | Flow: Authorize

[x] Request parity
    Evidence: Shadow traffic log 2026-03-20, 10K requests compared, 0 discrepancies
[x] Response mapping
    Evidence: Unit test test_hdfc_authorize_all_statuses passes
[x] Error handling
    Evidence: Tested with expired API key, gateway 500, malformed JSON
[x] Authentication
    Evidence: Sandbox test with real HDFC credentials
[x] Encryption
    Evidence: AES encryption verified, matches euler_txn output byte-for-byte
[x] Redirect flows
    Evidence: 3DS enrollment -> redirect URL matches old path
[ ] Webhook parsing
    Note: Not applicable for authorize flow
[x] Edge cases
    Evidence: Zero-auth, partial capture, multi-currency all tested
[x] Performance
    Evidence: p50=180ms (old=175ms), p99=450ms (old=420ms)
[x] Timeout handling
    Evidence: 30s timeout configured, tested with slow gateway response
```
