# 7.2 Code Pattern Translation Guide

> **Reference — Haskell to Rust translation patterns**
>
> Use this guide when translating euler_txn gateway adapter code into
> connector-service connector implementations.

## 9.1 Gateway Adapter: Haskell -> Rust

### Request Building

**euler_txn pattern (Haskell):**

```haskell
-- Gateway/<Name>/Transforms.hs
make_<Gateway>AuthRequest :: OrderReference -> TxnDetail -> MerchantGatewayAccount -> ... -> <Gateway>AuthRequest
make_<Gateway>AuthRequest order txn mga ... = <Gateway>AuthRequest
  { merchantId    = mga ^. _merchantId
  , amount        = show $ order ^. _amount
  , currency      = fromMaybe "INR" $ order ^. _currency
  , orderId       = txn ^. _orderId
  , customerEmail = order ^. _customerEmail
  , returnUrl     = ...
  }

-- Gateway/<Name>/Flow.hs
initiateTxn :: ... -> BackendFlow st Configs PaymentResp
initiateTxn ... = do
  let request = make_<Gateway>AuthRequest order txn mga ...
  let url     = baseUrl <> "/api/v1/pay"
  let headers = [("Authorization", "Bearer " <> mga ^. _apiKey)]
  response <- callGatewayAPI url headers (encode request)
  case decode response of
    Right resp -> mapToPaymentResp resp
    Left err   -> throwErr GATEWAY_ERROR
```

**connector-service equivalent (Rust):**

```rust
// connectors/<name>/transformers.rs
#[derive(Serialize)]
pub struct <Name>PaymentsRequest {
    merchant_id: String,
    amount: String,
    currency: String,
    order_id: String,
    customer_email: String,
    return_url: String,
}

impl TryFrom<&RouterDataV2<Authorize, PaymentFlowData, PaymentsAuthorizeData, PaymentsResponseData>>
    for <Name>PaymentsRequest
{
    type Error = Report<ConnectorError>;
    fn try_from(item: &RouterDataV2<...>) -> Result<Self, Self::Error> {
        Ok(Self {
            merchant_id: item.resource_common_data.merchant_id.clone(),
            amount: item.request.amount.to_string(),
            currency: item.request.currency.to_string(),
            order_id: item.request.order_id.clone(),
            customer_email: item.request.email.clone().unwrap_or_default(),
            return_url: item.request.return_url.clone().unwrap_or_default(),
        })
    }
}

// connectors/<name>/mod.rs
impl ConnectorIntegrationV2<Authorize, PaymentFlowData, PaymentsAuthorizeData, PaymentsResponseData>
    for <Name>
{
    fn get_url(&self, req: &RouterDataV2<...>) -> CustomResult<String, ConnectorError> {
        Ok(format!("{}/api/v1/pay", self.base_url(req.resource_common_data.connectors)?))
    }

    fn get_headers(&self, req: &RouterDataV2<...>) -> CustomResult<Vec<(String, Maskable<String>)>, ConnectorError> {
        let auth = <Name>AuthType::try_from(&req.connector_config.connector_auth)?;
        Ok(vec![("Authorization".to_string(), format!("Bearer {}", auth.api_key).into())])
    }

    fn get_request_body(&self, req: &RouterDataV2<...>) -> CustomResult<Option<RequestContent>, ConnectorError> {
        let body = <Name>PaymentsRequest::try_from(req)?;
        Ok(Some(RequestContent::Json(Box::new(body))))
    }

    fn handle_response_v2(&self, data: &RouterDataV2<...>, res: Response)
        -> CustomResult<RouterDataV2<...>, ConnectorError>
    {
        let response: <Name>PaymentsResponse = res.response.parse_struct("<Name>PaymentsResponse")?;
        // Map gateway response -> unified PaymentsResponseData
        // CRITICAL: Status mapping must match euler_txn's existing behavior exactly
    }
}
```

### Key Translation Rules

| Haskell Pattern | Rust Equivalent |
|----------------|----------------|
| `mga ^. _apiKey` (lens access) | `req.connector_config.connector_auth` -> parse into auth type |
| `order ^. _amount` | `item.request.amount` |
| `fromMaybe "INR" $ order ^. _currency` | `item.request.currency.to_string()` (or `.unwrap_or(Currency::INR)`) |
| `encode request` (Aeson JSON) | `RequestContent::Json(Box::new(body))` |
| `decode response` (Aeson JSON) | `res.response.parse_struct("TypeName")` |
| `throwErr GATEWAY_ERROR` | `Err(ConnectorError::...)` with `error_stack::Report` |
| `BackendFlow st Configs a` | `CustomResult<T, ConnectorError>` |

## 9.2 Response Status Mapping: Haskell -> Rust

**euler_txn pattern:**

```haskell
-- Gateway/<Name>/Flow.hs
mapResponseToStatus :: <Gateway>Response -> TxnStatus
mapResponseToStatus resp = case resp ^. _status of
  "SUCCESS"    -> CHARGED
  "PENDING"    -> AUTHORIZING
  "FAILED"     -> AUTHORIZATION_FAILED
  "CANCELLED"  -> VOIDED
  _            -> PENDING
```

**connector-service equivalent:**

```rust
// connectors/<name>/transformers.rs
impl From<<Name>PaymentStatus> for AttemptStatus {
    fn from(status: <Name>PaymentStatus) -> Self {
        match status {
            <Name>PaymentStatus::Success   => Self::Charged,
            <Name>PaymentStatus::Pending   => Self::Pending,
            <Name>PaymentStatus::Failed    => Self::Failure,
            <Name>PaymentStatus::Cancelled => Self::Voided,
        }
    }
}
```

### Critical Rule

**The Rust `From` implementation must produce the exact same mapping as the Haskell `mapResponseToStatus` function.** If euler_txn maps gateway status `"PENDING"` to `AUTHORIZING`, then connector-service must map it to `Pending` (which is the equivalent — see [7.1_type_mapping.md](7.1_type_mapping.md) section 7.1.1).

## 9.3 Error Mapping: Haskell -> Rust

**IMPORTANT: `ResponseRouterData` Pattern**

`ResponseRouterData` has exactly **2 type parameters** (defined in `backend/connector-integration/src/types.rs:109`):

```rust
pub struct ResponseRouterData<Response, RouterData> {
    pub response: Response,      // Connector-specific response (e.g., PhonepePaymentsResponse)
    pub router_data: RouterData, // The full RouterDataV2<Flow, CommonData, Req, Resp>
    pub http_code: u16,
}
```

The response transformer `TryFrom` impl pattern:

```rust
impl<T: PaymentMethodDataTypes + std::fmt::Debug + Sync + Send + 'static + Serialize>
    TryFrom<ResponseRouterData<ConnectorPaymentsResponse, Self>>   // <-- exactly 2 type params
    for RouterDataV2<Authorize, PaymentFlowData, PaymentsAuthorizeData<T>, PaymentsResponseData>
{
    type Error = Error;

    fn try_from(item: ResponseRouterData<ConnectorPaymentsResponse, Self>) -> Result<Self, Self::Error> {
        let response = &item.response;
        // Access fields via: item.response, item.router_data, item.http_code
        Ok(Self {
            response: Ok(PaymentsResponseData::TransactionResponse { ... }),
            resource_common_data: PaymentFlowData {
                status: AttemptStatus::Charged,
                ..item.router_data.resource_common_data
            },
            ..item.router_data
        })
    }
}
```

**Common mistakes to avoid:**
- Do NOT use 5 type params like `ResponseRouterData<Flow, Response, CommonData, Req, Resp>` — it only takes 2.
- The second param is `Self` (the `RouterDataV2` being implemented for).
- Access the original router data via `item.router_data` (NOT `item.data`).
- Use `..item.router_data` spread to carry forward unchanged fields.

---**euler_txn pattern:**

```haskell
-- Uses bankErrorCode + bankErrorMessage from gateway response
updateTxnWithError txn errorCode errorMessage = do
  DB.updateTxnDetail txn { bankErrorCode = Just errorCode, bankErrorMessage = Just errorMessage }
```

**connector-service equivalent:**

```rust
// In handle_response_v2 or get_error_response:
ErrorResponse {
    status_code: res.status_code,
    code: error.code.clone(),           // maps to bankErrorCode
    message: error.message.clone(),      // maps to bankErrorMessage
    reason: error.reason.clone(),
    attempt_status: None,
}
```

### Error Field Mapping

| euler_txn Field | connector-service Field |
|----------------|----------------------|
| `bankErrorCode` | `ErrorResponse.code` |
| `bankErrorMessage` | `ErrorResponse.message` |
| HTTP status code | `ErrorResponse.status_code` |

## 9.4 Bridge Call Pattern (euler_txn -> connector-service)

This is the existing pattern used for Razorpay/PhonePe UPI in production.

**Source:** `euler-x/src-generated/Gateway/ConnectorService/Flow.hs`

```haskell
-- 1. Build unified request body
reqBody <- Common.buildUnifiedSdkParamsOrSendCollectReqBody ord txn txnCardInfo ...

-- 2. Build connector-service URL + auth headers
(url, headers) <- build<Gateway>Headers mga connectorServiceBaseUrl endpoint

-- 3. Make HTTP call to connector-service
response <- makeConnectorServiceRequest url headers reqBody apiTag

-- 4. Parse connector-service response
case A.decode response of
  Just csResponse -> convertConnectorResponseToSdkParams gateway csResponse ...
  Nothing -> throwErr CONNECTOR_SERVICE_PARSE_ERROR

-- 5. Map back to euler_txn types
paymentResp <- mapCsResponseToPaymentResp csResponse txnDetail orderRef
```

### Headers to Pass

```haskell
headers =
  [ ("x-connector",   gatewayName)        -- e.g., "razorpay"
  , ("x-auth",        mga ^. _authType)   -- e.g., "HeaderKey"
  , ("x-api-key",     mga ^. _apiKey)     -- gateway API key
  , ("x-key1",        mga ^. _key1)       -- additional secret
  , ("x-merchant-id", mga ^. _merchantId) -- gateway merchant ID
  , ("Content-Type",  "application/json")
  ]
```

## 9.5 Common Type Translations

### Amount Handling

```haskell
-- euler_txn: amount is Rational or Double, often shown as string
show $ order ^. _amount        -- "100.00"
```

```rust
// connector-service: amount is i64 in minor units (paise/cents)
item.request.amount             // 10000 (for Rs 100.00)
```

**The bridge must convert:** `amount * 100` when going from euler_txn to connector-service (if euler_txn uses major units).

### Optional Fields

```haskell
-- euler_txn: Maybe types
fromMaybe defaultVal $ field
```

```rust
// connector-service: Option types
field.unwrap_or(default_val)
// or
field.ok_or(ConnectorError::MissingRequiredField { field_name: "field" })?
```

### Record Construction

```haskell
-- euler_txn: Haskell record syntax
<Gateway>AuthRequest { field1 = val1, field2 = val2, ... }
```

```rust
// connector-service: Rust struct init
<Name>PaymentsRequest {
    field1: val1,
    field2: val2,
    // ...
}
```

### Sum Types / Enums

```haskell
-- euler_txn: Haskell algebraic data types
data TxnFlowType = CARD_TRANSACTION | NET_BANKING | INTENT | COLLECT | ...
```

```rust
// connector-service: Rust enums
pub enum PaymentMethodData {
    Card(CardData),
    BankRedirect(BankRedirectData),
    Upi(UpiData),
    // ...
}
```

Note: connector-service enums often carry data (are not just simple variants like euler_txn).
